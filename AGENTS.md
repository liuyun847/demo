- **项目类型**: 游戏项目（bilibili游戏大赛参赛项目）
- **当前阶段**: 初始化阶段，尚未实现具体业务功能

# 项目结构
```
demo/
├── .godot/                    # Godot引擎缓存目录（自动生成，无需提交）
│   ├── editor/                # 编辑器配置缓存
│   ├── imported/              # 资源导入缓存
│   ├── shader_cache/          # 着色器编译缓存
│   └── 其他缓存文件
├── icon.svg                   # 项目图标
├── icon.svg.import            # 图标导入配置
├── project.godot              # Godot项目核心配置文件
├── .editorconfig              # 编辑器代码风格配置
├── .gitattributes             # Git属性配置
└── .gitignore                 # Git忽略文件配置
```

# 运行方式
## 编辑器运行
1. 安装Godot 4.6版本
2. 打开Godot编辑器，导入本项目根目录
3. 点击编辑器右上角的「运行」按钮即可启动项目

## 命令行运行
```powershell
godot --path "c:\Users\MLTZ\Desktop\程序\godot\bili游戏大赛\demo"
```

# 开发规范
1. **编码语言**: GDScript，遵循官方GDScript编码规范
2. **文件编码**: 所有代码文件使用UTF-8编码
3. **注释规范**: 代码注释使用中文，优先添加类型注释、函数/类文档说明，关键逻辑添加内联注释
4. **资源管理**: 所有游戏资源存放在对应分类目录下，保持项目结构清晰
5. **版本控制**: 提交代码前确保.gitignore配置正确，避免提交引擎缓存文件

# 测试与验证
- 项目开发阶段，使用Godot编辑器内置调试功能进行功能测试
- 功能完成后，可使用`godot --export`命令导出对应平台的安装包进行整体测试

# 部署方式
- Windows平台：使用Godot编辑器导出Windows可执行文件
- 其他平台：根据需求使用Godot对应的导出模板进行打包
